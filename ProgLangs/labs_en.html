Practice exercises

Exercise 1

Create the methods conjugate() and reciprocate() and div() in class Complex.

  • The conjugate of \(a + bi = a - bi\)
  • The reciprocate of \(a + bi = {a \over a^2 + b^2} - {b \over a^2 + b^2}i\)
  • Division works like this.

\[\frac{a + bi}{c + di} = \frac{(a + bi)(c - di)}{(c + di)(c - di)} = \frac{(ac + bd) + (bc - ad)i}{c^2 + d^2} = \frac{ac + bd}{c^2 + d^2} + \frac{bc - ad}{c^2 + d^2}i\]

Exercise 1

Following MultiDimensionalPointSuite, create a point that can have any number of dimensions. Its constructor takes its initial coordinates. The methods get and set operate on individual indexes.

The class implements two interfaces.

  • Implement the method of Scalable. It multiplies each coordinate by a number.
  • For Undoable, your task is more complex.
    • You need to be able to undo the effect of all methods that change the point’s coordinates.
    • Therefore, the lastCoordinates field is always filled with the previous state of the point.
      • Upon creation, fill it with the initial coordinates.
    • The interface’s method restores the last state.
      • If the method is called again immediately, the state that was replaced just now is restored.

Test the following. In the following test cases, take the following steps.

  1. Create a point.
  2. Perform the described action.
  3. For each coordinate, check that it contains the expected value.
  4. Call undo.
  5. For each coordinate, check that it contains the value after creation.
  6. Call undo again.
  7. For each coordinate, check that it contains the expected value again.

The scenarios are the following.

  • testInitUndo: just create a point
  • testSet: create a point, then set one of its coordinates
  • testScale: create a point, then scale it

Let two points be equal to each other if their coordinates exactly match, ignoring their histories.

In method testEquality, test that if you create two lists of points that are pairwise equal to each other, then the two lists are equal to each other.

Also define the natural comparison of two points as follows: if one has fewer dimensions, that one is smaller; otherwise, the first coordinate they differ in determines which is smaller; if all their coordinates match, they are equal.

In method testOrdering, put lots of points into a list. Create them with various dimensions and values. Then invoke Collections.shuffle on the list followed by Collections.sort. After that, test that the elements of the list are in the expected increasing order.
